Congratulations you got it!

It seems that the insane PHP framework «tsFramework» works on your webserver! Let's start creating your website.

You need help for that? No problem! Visit the website of this framework, write into the forums and read guides in the wiki.

You'll profit from five major advantages during your development but also afterwards.

  1. Unbeatable fast

    The «tsFramework» has one in mind: speed. Its light architecture and optimized PHP code programmed by experienced web developers are a good foundation for reaching that aim.

    But for best performance there is also server sided optimization needed. The best PHP script works sluggish if the web server is configured bad. So check your server configuration! Some good advices for tuning your machine you can find here.

  2. Insane flexible

    Blog, Forum, CMS, e-shop... everything no problem for the «tsFramework». You can use this framework for almost every task you face in daily web development business.

    For some requirements there are propably already some modules you can just download and install on top of your base framework installation. Adapting them to your use should be easy afterwards.

  3. Cheap as hell

    «tsFramework» is scripted in PHP. PHP is supported by more or less every webspace provider and is a webwide standard. PHP web hostings are cheap to get (there are also some for free).

    PHP and the database MySQL are open source and free available. Also the most extensions for this two technologies.

  4. Full control

    The framework itself is very small and only comes with a "page delivery" module and some core functions like multilingual support and so on. But it also provides uncountable other modules which you can put on top of the framework and extend it's funcitonality if needed. (e.x. database support, form valdation, facebook API, etc.) So you have always the full controll over the components and the ability of your framework installation.

  5. Custom Tags

    JSF is bullshit, we all know that. But there is one thing which nobody will miss in PHP which JSF has: Custom tags. Well PHP has it now too. (At least the «tsFramework».)

    Now you can create your own HTML tags which include for e.x. program logic and render a specific component like a HTML table (already filled with data from - let's say - a database) or something like that.
